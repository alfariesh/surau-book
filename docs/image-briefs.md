# Image Briefs

Status: `parked_experimental`

Image briefs are a reviewable visual-planning layer for chapter separators and other non-canonical book design assets.

They do not change Arabic text, passage IDs, citations, semantic annotations, or translations. They only describe what kind of image could be generated or sourced later.

This layer is intentionally parked for now. Keep it as a future enhancement note; do not generate image assets or wire images into Typst editions until the core kitab-processing workflow, editorial review, citation, translation, and API work are stable.

## Flow

```text
clean/manuscript.md
  + passages.jsonl
  + semantic annotations
  -> image-briefs.jsonl
  -> human review
  -> parked until visual direction is approved
```

## File Location

```text
books/{work_id}/assets/image-briefs.jsonl
```

Generated assets should later live under:

```text
books/{work_id}/assets/images/
```

Do not create or commit generated image assets during the current core-processing phase.

## Guardrails

Allowed:

- abstract Islamic geometry
- manuscript texture
- ink, paper, margin ornament, illumination
- symbolic diagrams, networks, timelines, maps
- architecture or objects when not making historical claims
- chapter mood and theme

Avoid:

- depiction of Prophet Muhammad
- depiction of companions, angels, Jibril, divine beings
- literal heaven, hell, resurrection, unseen realities
- fake historical scenes
- pseudo-Arabic calligraphy pretending to be sacred text
- photorealistic people
- stock-photo aesthetics
- visual claims that imply hadith grading, sanad authenticity, or historical certainty

## Placement Policy

Current policy: no active placement in generated editions.

Start with chapter separators only:

```text
placement = chapter_separator
```

Recommended density:

- one visual for each major H1/H2 separator during prototype
- fewer images for dense formula sections
- no images inside canonical matn blocks unless later approved by editorial policy

## Prompt Language

Prompts should be written in English because most image models follow English visual instructions more reliably.

Keep Arabic section titles as metadata. Do not ask the image model to render Arabic text unless a reviewed calligraphy asset is provided separately.

## Review Status

```text
draft       generated by script or LLM
reviewed    approved image brief, still no final asset
generated   asset created from brief
approved    asset approved for layout
rejected    should not be used
```

Only `approved` image assets should be included in a published print edition.

For now, keep all rows as `draft` or `parked`. The active pipeline should ignore image briefs.
